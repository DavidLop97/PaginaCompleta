Guía de estilo HTML

Un código HTML coherente, limpio y ordenado hace que sea más fácil para otros leer y comprender su código.

A continuación se presentan algunas pautas y consejos para crear un buen código HTML:

Declarar siempre el tipo de documento

Declare siempre el tipo de documento como la primera línea de su documento.

El tipo de documento correcto para HTML es:

<!DOCTYPE html>

Utilice nombres de elementos en minúsculas

HTML permite mezclar letras mayúsculas y minúsculas en los nombres de elementos, pero recomendamos usar nombres en minúsculas por varias razones:

  • Mezclar nombres en mayúsculas y minúsculas tiene mala pinta.
  • Los desarrolladores normalmente usan nombres en minúsculas.
  • Las minúsculas son más limpias y fáciles de escribir.

Ejemplos:

<body>
              <p>This is a paragraph.</p>
            </body>

Utilice nombres de atributos en minúsculas

Al igual que con los nombres de elementos, es mejor usar nombres de atributos en minúsculas por consistencia y legibilidad:

<a href="https://www.w3schools.com/html/">Visit our HTML tutorial</a>

Siempre citar valores de atributos

Es una buena práctica citar siempre los valores de los atributos, especialmente cuando contienen espacios o caracteres especiales:

<table class="striped">

Especifique siempre alt, ancho y alto para las imágenes

Para imágenes, siempre especifique el atributo alt y los atributos de ancho y alto para mejorar la accesibilidad y evitar problemas de diseño:

<img src="html5.gif" alt="HTML5" style="width:128px;height:128px">

Espacios y signos iguales

Es recomendable mantener espacios alrededor de los signos de igual para mejorar la legibilidad:

<link rel="stylesheet" href="styles.css">

Buen ejemplo de tabla

<table>
              <tr>
                <th>Name</th>
                <th>Description</th>
              </tr>
              <tr>
                <td>A</td>
                <td>Description of A</td>
              </tr>
              <tr>
                <td>B</td>
                <td>Description of B</td>
              </tr>
            </table>

Buen ejemplo de lista

<ul>
              <li>London</li>
              <li>Paris</li>
              <li>Tokyo</li>
            </ul>
